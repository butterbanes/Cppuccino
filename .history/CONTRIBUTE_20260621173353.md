# Contributing to Cppuccino

Thank you for your interest in contributing to Cppuccino! Whether you are reporting a bug, improving the documentation, or adding a new primitive wrapper, we appreciate your support.

## Getting Started

1. Fork the repository and create your branch from main.
2. Setup your environment: Ensure you have cmake and a C++20-compatible compiler (like g++ 13+) installed.
3. Build the project:

   ```sh
   mkdir build && cd build
   cmake ..
   make

   # OPTIONAL: run the following to add the headers to your system
   sudo make install
   ```

## Workflow & Coding Standards

* Modern C++: Please use C++20 standards. We prefer {} (brace) initialization over () for object creation to prevent narrowing conversions.
* Synchronization: If you add a new primitive wrapper (e.g., Long), you must add the corresponding header in include/Cppuccino/ and the implementation in src/.
* Testing: Every new feature must be accompanied by a unit test in tests/. We use Google Test.

## Running Tests

To verify your changes, run the test suite (if the respective suite exists, if not feel free to help create them):

# Inside your build directory

cmake -DBUILD_TESTING=ON ..
make
./tests/Cppuccino_unit_test

Note: If you are only testing the library and not the test suite, you can toggle testing off to speed up build times.

## Pull Request Process

1. Keep it focused: One Pull Request (PR) per feature or bug fix.
2. Update Tests: Ensure all existing tests pass and add new tests for your changes.
3. Documentation: If you add a new class, please update the README to reflect its usage.
4. Commit Policy: Keep commit messages descriptive (e.g., "Add Byte wrapper implementation").
